INSTRUCCIÓNS:
(TEÑEN QUE EXISTIR OS NOMES, SE NON OBVIAMENTE QUE NON FUNCIONA)
M -> son as funcións qeu necesitan previamente ter feito un allocate -malloc para poder facer as funcións correspondentes.

ALLOCATE (complicada)
allocate -> mostra lista de bloques asignados para o proceso (todos)
allocate -malloc -> mostra lista de bloques asignados que son de tipo malloc
allocate -malloc tam -> asigna un bloque de memoria de tamaño tam de tipo malloc
allocate -createshared -> mostra lista de bloques asignados que son de tipo shared
allocate -createshared cl -> mostra lista de bloques asignados que son de tipo shared
allocate -createshared tam -> mostra lista de bloques asignados que son de tipo shared
allocate -createshared cl tam -> asigna memoria compartida de nombre de clave cl e de tamaño tam
allocate -shared -> mostra lista de bloques asignados que son de tipo shared
allocate -shared cl -> asigna bloque de memoria xa existente de clave cl
allocate -mmap -> mostra lista de bloques asignados que son de tipo mmap
allocate -mmap fich -> error (Imposible mapear ficheiro)
allocate -mmap perm -> error (Imposible mapear ficheiro)
allocate -mmap fich perm -> mapea un ficheiro de nombre fich con permisos perm
allocate diferente -> mostrase unha mini axuda

DEALLOCATE (complicada) 
deallocate -> mostra lista de bloques asignados para o proceso (todos)
deallocate -malloc -> mostra lista de bloques asignados para o proceso (malloc)
deallocate -malloc tam -> desasigna o primeiro bloque malloc de tamaño tam
deallocate -shared -> mostra lista de bloques asignados para o proceso (shared)
deallocate -shared cl -> elimina bloque de memoria compartida cl
deallocate -delkey -> mostra lista de bloques asignados para o proceso (shared)
deallocate -delkey cl -> elimina memoria do bloque de memoria compartida cl
deallocate -mmap -> mostra lista de bloques asignados para o proceso (mmap)
deallocate -mmap fich -> desmapease o ficheiro fich que foi mapeado anteriormente (que se fixo allocate -mmap fich perm)
deallocate addr -> desasignase o bloque de memoria en addr.
deallocate diferente -> lista de todos os bloques asignados 

RECURSE (fácil)
recurse -> non fai nada
recurse n -> fai a función recursiva n veces
recurse diferente -> fai a recursiva solo 0 veces (solo aparece unha liña a que se mostra)

I-O (fácil-regular, a metade xa esta feita dos profes, modificar pouco desa función) M
i-o -> fai como unha mini axuda
i-o read -> pon que faltan parametros
i-o read fich -> pon que faltan parametros
i-o read addr -> pon que faltan parametros
i-o read cont -> pon que faltan parametros
i-o read fich addr -> pon que faltan parametros
i-o read fich cont -> pon que faltan parametros
i-o read addr cont -> pon que faltan parametros
i-o read fich addr cont -> debería leer cont bytes dende addr a fich
i-o write -> pon que faltan parametros
i-o write fich -> pon que faltan parametros
i-o write addr -> pon que faltan parametros
i-o write cont -> pon que faltan parametros
i-o write fich addr -> pon que faltan parametros
i-o write fich cont -> pon que faltan parametros
i-o write addr cont -> pon que faltan parametros
i-o write fich addr cont -> debería escribir cont bytes dende addr a fich
i-o write -o -> pon que faltan parametros
i-o write -o fich -> pon que faltan parametros
i-o write -o addr -> pon que faltan parametros
i-o write -o cont -> pon que faltan parametros
i-o write -o fich addr -> pon que faltan parametros
i-o write -o fich cont -> pon que faltan parametros
i-o write -o addr cont -> pon que faltan parametros 
i-o write -o fich addr cont -> debería sobreescribir cont bytes dende addr a fich
i-o diferente -> se non pos ningunha das opcións, ensinache unha mini axuda

MEMDUMP (regular) M
memdump -> non fai nada 
memdump addr -> daría unha violación de segmento
memdump cont -> daría unha violación de segmento
memdump addr cont -> debería volcar en pantalla os contidos na posición de memoria addr con cont bytes
memdump diferente -> non se sabe (debería de mostrar error)

MEMFILL (regular) M
memfill -> non fai nada 
memfill addr -> *
memfill cont -> daría unha violación de segmento
memfill byte -> daría unha violación de segmento
memfill addr cont -> *
memfill addr byte -> daría que se enche 0 bytes.
memfill cont byte -> daría unha violación de segmento
memfill addr cont byte -> * (supuestamente debería encher a memoria a pratir da dirección addr con cont (numero de bytes) de byte)
memfill diferente -> non se sabe
* -> habería que poñer unha situación que sempre sería a mesma (rollo se śe fai memfill 0x412256 (sendo esta unha dirección válida), que sempre se añade o mesmo caracter n veces, é decir, definir o caracter e mais as veces que se quere imprimir))

MEMORY (fácil)
memory -> mostra toda a información (excepto pmap)
memory -blocks -> mostra a lista de bloques 
memory -funcs -> mostra os bloques de memoria da funcións do programa e mais as da libreria
memory -vars -> mostra os bloques onde estan as variables locales, as variables globales e mais as variables estaticas.
memory -pmap -> mostra saída comando pmap (que se fai sobre o executable) (foto no móvil)
memory -all -> mostra toda a información que aportan blocks, funcs e vars.
memory diferente -> ponse que non se acepta a opción diferente
